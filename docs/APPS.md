## 開発ツール

### エディタ

| アプリ | 説明 | 選定理由 |
|--------|------|----------|
| **Cursor** | AI搭載コードエディタ | VS Code互換でAI機能統合。Claude/GPT-4でコード生成・編集 |
| **Visual Studio Code** | 汎用コードエディタ | 拡張機能エコシステム、デバッグ、Git統合。業界標準 |
| **MacVim** | Vim GUIアプリ | システムクリップボード統合、macOSネイティブUI |

### なぜCursorか？

```
選択肢: Cursor vs GitHub Copilot (VS Code) vs Windsurf vs Zed

Cursor:
  ✓ VS Code完全互換（設定、拡張機能そのまま）
  ✓ AI機能がエディタにネイティブ統合
  ✓ コンテキスト認識が優秀（プロジェクト全体を理解）
  ✓ Claude対応
  ✓ 無料枠あり

GitHub Copilot:
  ✓ VS Code統合
  ✗ 拡張機能として動作（ネイティブ統合ではない）

Zed:
  ✓ 超高速（Rust製）
  ✗ 拡張機能エコシステムが発展途上

→ Cursorを採用：AI開発体験が最も優れている
  （VS Codeも互換性のため併用）
```

### コンテナ & 仮想化

| アプリ | 説明 | 選定理由 |
|--------|------|----------|
| **Docker Desktop** | Dockerコンテナ環境 | 公式ツール、Kubernetes統合、GUIダッシュボード |
| **OrbStack** | 軽量Docker/Linux環境 | Docker Desktopより高速・軽量。代替として推奨 |

### なぜ両方インストールするか？

```
Docker Desktop:
  ✓ 公式サポート
  ✓ Docker Compose V2
  ✓ Kubernetes統合
  ✗ メモリ消費が多い
  ✗ 起動が遅い

OrbStack:
  ✓ 高速（起動2秒）
  ✓ 軽量（メモリ効率良い）
  ✓ Linux VM機能
  ✓ ファイル共有が高速
  ✗ 一部互換性の問題あり

→ 両方インストール：OrbStackをメインに、互換性問題時にDocker Desktop使用
```

## ターミナル

| アプリ | 説明 | 選定理由 |
|--------|------|----------|
| **Ghostty** | モダンターミナル | GPU加速、ネイティブUI、高速、設定シンプル |
| **Warp** | AI搭載ターミナル | コマンド補完、履歴検索、チーム共有機能 |

### なぜGhosttyか？

```
選択肢: Ghostty vs iTerm2 vs Alacritty vs Kitty vs Warp

Ghostty:
  ✓ ネイティブmacOS UI（メニューバー、タブ）
  ✓ GPU加速で高速レンダリング
  ✓ Zig製で軽量
  ✓ シンプルな設定ファイル
  ✓ 開発者（Mitchell Hashimoto - HashiCorp創設者）の信頼性

iTerm2:
  ✓ 機能豊富
  ✗ Electron製ではないが重め
  ✗ 設定が複雑

Alacritty:
  ✓ 高速
  ✗ タブ機能なし（tmux必須）
  ✗ macOS統合が弱い

Warp:
  ✓ AI機能
  ✗ アカウント必須
  ✗ プライバシー懸念

→ Ghosttyをメインに採用：速度、機能、macOS統合のバランスが最良
  Warpはサブとして使い分け
```

|--------|------|----------|

| **Raycast** | モダンランチャー | 無料、拡張機能ストア、スクリプト実行 |

### なぜ両方か？

```
Alfred:
  ✓ 長年の実績と安定性
  ✓ Powerpack（有料）でWorkflowが強力
  ✓ クリップボード履歴が優秀

Raycast:
  ✓ 無料で高機能
  ✓ モダンなUI
  ✓ 拡張機能ストアが充実
  ✓ 開発者向け機能

→ 好みで選択可能。Alfredの有料版を持っていればAlfred、
  無料で始めるならRaycast
```

## アプリ選定の原則

1. **ネイティブ優先**: Electronより可能な限りネイティブアプリを選択

2. **オープンソース重視**: 可能な限りOSSを選択（VLC, LibreOffice等）

3. **統一感**: テーマ、フォント、UIを統一（Catppuccin, JetBrains Mono）

4. **必要十分**: 過剰なアプリを入れない、使うものだけ

5. **代替可能性**: 複数の選択肢がある場合は両方インストールし、使い比べ
